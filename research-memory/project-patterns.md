# Project Patterns

Purpose: transferable lessons from completed or active projects. Keep project-specific facts in each project folder; keep only reusable patterns here.

## Reusable Patterns

### Protected Area Human-Pressure Edge

- Treat PA boundaries as gradients rather than binary lines.
- Use "human-pressure edge", "protective halo", and "pressure-concentration edge" before using "leakage".
- Separate static edge slopes from dynamic trend slopes.
- Keep inherited sample filters under review when manuscript framing changes.

### Global Spatial Workflows

- Run pilot diagnostics first.
- Preserve old date-tagged outputs when changing sample universe or metric definitions.
- Log unresolved computational exclusions and decide whether they are acceptable.

## Candidate Patterns To Validate

- Whether edge-depth estimates are stable under 80/90/95% thresholds.
- Whether trend-based and static-based classifications should be co-primary or primary/auxiliary.
